This assignment introduces crucial techniques for handling "messy" real-world data. Real data is rarely perfect; it has missing chunks and absurd errors (outliers). This code demonstrates how to clean those errors mathematically without destroying the dataset.

Here is a complete breakdown of the code, an interactive tool to understand the math, and the materials you need for your viva.

### **1. Core Concepts & Step-by-Step Explanation**

* **Step 1: Dataset Creation**
* **Concept:** We create a mock dataset (`df`) using a dictionary. We intentionally inject problems: `np.nan` (missing values) and unrealistic numbers (like a Math Score of 150 out of 100, or a Reading Score of 10).


* **Step 2: Missing Value Imputation (The Median)**
* **Concept:** The code finds the missing Math scores and fills them.
* **Why Median?** Usually, people fill missing gaps with the "Mean" (Average). However, the Mean is highly sensitive to outliers. Because we have a score of 150, the Mean would be artificially dragged upward. The **Median** (the exact middle value when sorted) completely ignores extreme highs or lows, making it a much safer guess.


* **Step 3: Handling Outliers (The IQR Method)**
* **Concept:** IQR stands for Interquartile Range. It represents the middle 50% of your data (where the "normal" data lives).
* **The Math:**
* Find Q1 (25th percentile) and Q3 (75th percentile).
* Calculate the IQR = Q3 - Q1.
* Establish a "safe zone" by calculating a Lower Bound ($Q1 - 1.5 \times IQR$) and an Upper Bound ($Q3 + 1.5 \times IQR$).


* **Capping (`np.where`):** Instead of deleting the row with the 150 score (which would lose the student's valid reading/writing scores), `np.where` acts like a filter. It says: "If a score is higher than the Upper Bound, change it to equal the Upper Bound." This brings extreme values back to the edge of normality.


* **Step 4: Log Transformation**
* **Concept:** Machine Learning models assume data is "Normally Distributed" (a symmetrical bell curve). If data is skewed (bunched up on one side with a long tail on the other), models struggle.
* Applying `np.log1p()` mathematically squashes the large extreme values closer together, pulling a skewed distribution into a shape that looks more like a healthy bell curve.

### **2. Potential Viva Questions & Answers**

**Q1: Why did you use the Median to fill missing values instead of the Mean?**

> **A:** The `Math_Score` column contained an extreme outlier (150). The mean is easily distorted by outliers, which would result in filling the missing values with an inaccurately high number. The median is robust against outliers.

**Q2: How does the IQR method identify an outlier?**

> **A:** It finds the middle 50% of the dataset (IQR = Q3 - Q1). It then defines boundaries by subtracting 1.5 times the IQR from Q1, and adding 1.5 times the IQR to Q3. Any point outside these boundaries is flagged as an outlier.

**Q3: In Step 3, you used `np.where()` to handle outliers. What exactly is this doing?**

> **A:** It is performing **Winsorization** or "Capping." Instead of deleting the entire row containing the outlier, `np.where` clips the extreme value, replacing it with the exact Upper or Lower bound limit. This preserves the rest of the student's valid data.

**Q4: Why do we perform Data Transformations like Log Transformation?**

> **A:** To reduce skewness. Many machine learning models (like Linear Regression) assume the data follows a Normal Distribution (Bell Curve). Log transformation compresses the long tail of right-skewed data, making its shape more normal.

**Q5: Why did you use `np.log1p()` instead of standard `np.log()`?**

> **A:** `np.log1p(x)` calculates `log(1 + x)`. If a dataset contains the value `0`, standard `np.log(0)` results in negative infinity, which will crash the code. Adding 1 ensures mathematical stability.

#### **Change Request 1: "Instead of IQR, use the Z-Score method to detect outliers."**

*Why they ask:* Z-score is the other major statistical method for outliers (best for normally distributed data).
*How to do it:*
Replace Step 3 entirely with this:

```python
from scipy import stats

def handle_outliers_zscore(dataframe, column_name):
    # Calculate absolute Z-scores
    z_scores = np.abs(stats.zscore(dataframe[column_name].dropna()))
    
    # Identify rows where Z-score is greater than 3 (Standard threshold)
    outliers = z_scores > 3
    print(f"Outliers found in {column_name}: {outliers.sum()}")
    
    # Keep only rows where Z-score is less than 3
    # Note: This deletes rows, unlike the IQR capping method
    dataframe = dataframe[(np.abs(stats.zscore(dataframe[column_name])) < 3)]
    return dataframe

df = handle_outliers_zscore(df, 'Math_Score')

```
